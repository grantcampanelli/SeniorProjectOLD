Adding a New Student

Continues from Common Step for All Tests
Case No. User Inputs and Commands Expected Output Remarks                   Results                  
4 Click on 'New Student' button under the 'Student' menu of the main navigation bar located at the top of the UI. 'New Student' Dialog opens; 'Student First Name', 'Student Last Name', 'Student ID', 'Student Nick Name', 'Student Group', 'Student Email', and 'Student Phone' fields contain placeholders 'i.e. John', 'i.e. Doe', 'i.e. 00000001', 'i.e. Johnny', 'i.e. "1" or "Node"', 'i.e. jdoe@gmail.com', and 'i.e. 123-456-7890', respectively. There are 'Save' and 'Cancel' buttons at the bottom right corner of the dialog. Testing the dialog load and composition. Pass
5 Click 'Save' with all fields empty. Error message 'Student first name can not be empty' displays at the bottom. Testing empty Student First Name validation. Pass
6 Click 'Save' with only the 'Student First Name' validly filled out but all other fields empty. Error message 'Student last name can not be empty' displays at the bottom. Testing empty Student Last Name validation. Pass
7 Click 'Save' with only the 'Student First Name' and 'Student Last Name' validly filled out but all other fields empty. Error message 'Student ID must be 8 digits' displays at the bottom. Testing empty Student ID. Pass
8 Click 'Save' with only the 'Student First Name', 'Student Last Name', and 'Student ID' validly filled out but all other fields empty. Error message 'Student email can not be empty' displays at the bottom. Testing empty Student Email. Pass
9 Click 'Save' with only the 'Student First Name', 'Student Last Name', 'Student ID', and 'Student Email' validly filled out but all other fields empty. Error message 'Student phone number must be 10 digits' displays at the bottom. Testing empty Student Phone. (All required fields have been covered at this point) Pass
10 Click 'Save' with all fields validly filled out except for the 'Student ID' (enter "aaaaaaaa" instead). Error message 'Student ID must be 8 digits' displays at the bottom. Testing Student ID entry validation. Pass
11 Click 'Save' with all fields validly filled out and selected except for the 'Student Phone' (enter "!@#$" instead). Error message 'Student phone number must be 8 digits' displays at the bottom. Testing Student Phone entry validation. Pass
12 Click 'Save' with all fields validly filled out and selected. No error message is displayed, and the dialog closes upon clicking on 'Save' button. Testing adding a valid Student to the list of students in a course. Pass
13 Refresh the browser (⌘R on Mac OS, F5 on Windows). Student that has been added is now showing on the list of students in the course (sorted alphabetically by last name). Testing update of list of students in a course after addition. Pass
12 Repeat step 4, and close the dialog using the 'x' button at the top right or the 'Cancel' button at the bottom right. The dialog closes and no changes are made to the list of students in a course. Testing closing and disposing the dialog. Pass
