Add a New Class

  1. Open Google Chrome
  2. Go to http://nodegrader.com/implementation/public/templates/addNewClassView.html
  3. Click the ‘Add New Class’ button to have the editable dialog box appear.
  4. Fill out the class information (Class Code and Number)
  5. Click ‘OK’ button to add a new class.

A dialog box should pop upon successful adding of a new class.

If all or part of the class information is missing, an alert pop up show up to instruct the user to enter all information.

Modify an Existing Class

  1. Open Google Chrome
  2. Go to http://localhost:63342/Grader/implementation/public/templates/modifyClassView.html
  3. Click ‘Modify Class Information’. This is a mockup for a trigger that a right-click event on a class title will trigger. An editable dialog box with current class information filled-in will appear.
  4. Make the desired changes to the class information.
  5. Click ‘OK’ button to apply the changes.

A dialog box should pop upon successful editing of the class information.

If all or part of the class information is missing, an alert pop up show up to instruct the user to enter all information.

REST Access to model data

  3. Go to http://app.nodegrader.com/api/Courses

The browser displays JSON  data showing all users and courses in the database

Set up an authenticated session with server

  1. Open Google Chrome
  2. Go to http://app.nodegrader.com
  3. Open the chrome developer tools window
  4. Go to the “Resources“ tab
  5. Expand “Session Storage” on the left side and click on “http://app.nodegrader.com” to see the local session data (there should be none).
  6. In the login form enter the username “admin” and password “admin”
  7. Submit the login form

After submitting the login form, the browser will authenticate with the server using a REST endpoint. The server will validate the credentials and send back a JSON object that defines a login property with the value true. The browser saves this object in session storage as shown in the developer tools window. Form that point on, the server will recognize the user in future requests made with the same cookie.

Develop client side infrastructure that drives the application. The backbone.js library provides model and view classes. The model classes expect its applications to use a REST api behind the scenes. The view classes expect its applications render html using templates like the one at “http://app.nodegrader.com/templates/error.hbs.” The application needs to be structured so that it effectively uses backbone.js.

  1. Go to “http://app.nodegrader.com/login”
  2. login with username: “admin” password: “admin”

When a user first connects to the application the browser begins a complex load process using require.js

The application loads dependencies like libraries, template files as it initializes the models and views. The login page is an example of how everything comes together. A backbone model object implements the session and a backbone view implements the form.

Add a category

  1. Open Google Chrome
  2. Go to http://nodegrader.com/implementation/public/templates/addCategoryView.html
  3. Click View->Developer->JavaScript Console in the Chrome menu bar
  4. Click the ‘Add Category’ button
  5. Fill out all the field in the Add Category dialog
  6. Click the ‘Ok’ button to submit the new category or the ‘Cancel’ button to close the dialog

A dialog box should pop up upon successfully adding a new category and the category information will be in the JavaScript console below.

Modify a category

  1. Open Google Chrome
  2. Go to http://nodegrader.com/implementation/public/templates/editCategoryView.html
  3. Click View->Developer->JavaScript Console in the Chrome menu bar
  4. Click the ‘Edit Category’ button
  5. Change any desired fields in the Edit Category dialog
  6. Click the ‘Ok’ button to submit the edited category or the ‘Cancel’ button to close the dialog

A dialog box should pop up upon successfully editing the category and the category information will be in the JavaScript console below.

Delete a category

  1. Open Google Chrome
  2. Go to http://nodegrader.com/implementation/public/templates/deleteCategoryView.html
  3. Click View->Developer->JavaScript Console in the Chrome menu bar
  4. Click the ‘Delete Category’ button
  5. Click the ‘Yes’ button to delete the category or the ‘No’ button to close the dialog

A dialog box should pop up upon successfully deleting the category and a message saying which button was clicked will be in the console below.
