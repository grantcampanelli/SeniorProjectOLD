Client Meeting: Grader Project Follow-Up

Discussion Topics and Corresponding Details

  • Questions and Answers
    Client's answers are in bold italics

    • Is this a website? There should be a desktop application for security reasons
    • Multiple class management
      • Can an instructor have multiple classes? Yes
      • Can a student be a part of multiple classes? Yes, similar to the class setting of CPE 308
      • Can multiple teachers be associated with a single class? Yes
    • Is the information on Grader available to public?
      • Only the class trend (statistics) information should bevisible to the public - no names or individual grades
    • How much information is visible to students?
      • Will out Grader anonoymously show individual assignment grades to students?
        • Technically yes, but it shouldn't be easy for students to guess whose grade is what and what their ID is based on this information.
    • Submission of classwork This is requirement is currently "pending"
      • Vision: Simple GUI for handin
        • Department server for the grading program
        • Our grader communicates with handin
        • Instructor can see who turned in what and when in our Grader
        • This will partially automate the late policy
    • Should the grader tool have an undo / redo feature?
      • Yes, it should, but it is a low priority feature
      • It may be difficult to implement, but it should be a standard feature for any tool
      • It should support up to 100 history items
    • Should updates be pushed to users in real-time?
      • No, instructor will work on grades and later "post" at a desired time.
      • Collaborative grading feature
        • This is an edge case
        • There cannot be multiple instructors equally in charge of a single class
        • Someone with update privillage should be able to lock the grading sesion
        • In conclusion, there will be no simultaneous collaborative grading feature
    • How does the weights / points / catogories work exactly?
      • Raw scores are for number of points for a test or a quiz, but not for a category
      • Relative or Absolute weight on categories
        • This should be a choice available to the user at the initial setup of a gradebook
          • Relative: weight is relative to the parent category
          • Absolute: weight is relative to the entire course grade
    • Drag-enabled curve setting feature
      • Is this curve setting visible to the students?
        • Students should be able to test out "what if"
        • Student should only be allowed to do this on his own grades
          • This should not affect instructor's view or other student's grade view
    • CSV data file Refer to the example posted on class site
      • Adding new left-column other than student names
        • Do you need to be able to add more columns?
          • These will be "information only" columns for information such as student's contact
      • Do you need to be able to export this data to a new CSV file?
        • This is an ideal but low priority feature
        • Importing from a raw file is also low priority feature
        • What matters is ability to interface with PeopleSoft and getting roster from it
